Concept & Core Gameplay

For Monster Escape mobile game the core concept is to ensure that the mobile game makes it a lot fun and engaging the users to come back to playing my own mobile game. I believe that the innovation gameplay, art style, narrative, multiplayer, and accessibility would make the mobile game unique. For the gameplay mechanics for Monster Escape is action-based challenges in which the player can interact the environment rapidly and move around left or right in a 2d environment until the player reach the goal at the end of each levels and defeat bosses at the end of the level of the world section the player is at. The game’s genre for Monster Escape is a 2d platformer and action game in which the game involves the player to run around, jump, and navigating levels.

Story & Themes

The mobile game for Monster Escape is that the game starts off with the adventurer who is going on a adventure to find some treasure in a castlethat is guarded by a bunch of enemies. The game is a 2d pixelated medieval styled themed game.

Target Audience

For the target audiences for Monster Escape mobile game, I believe that the mobile game Monster Escape is mostly going to mainly focus on casual gamers, teens, and adults. The reason why the game for Monster Escape is mainly focus on casual gamers, teens, and adults is because it allows gamers to play the game longer and the game consists of mild violence, mild language and suggestive themes.

Monetization Strategy

The game for Monster Escape will be monetized by making the game as an in-app purchases where the user will have to pay the premium for certain contents of the game like the supplies, gems, and unlimited lives and no ads and making the mobile game free-to-play with ads allowing the players to play the game but must watch ads if the player is not using premium.

Development Timeline

The process in creating a game for Monster Escape is to start with a mobile game pitch where you start putting ideas, drawing out the layout of the mobile game concepts, and also using the core mechanics for the game's control panels and also adding the UI, the assets, adding sprites for the characters enemies, items, obstacles, a boss enemy, adding animation, adding health either damage or healing, and also adding different background for each level. Then after I begin setting up Flow Lab and begin starting off with the game's environments, flow lab's built in codes, animation for the character, creating a scroll camera, and adding UI for the health bar and the coins collected for the first level. Then changing the assets for the background, the player, adding the boss enemy, adding more levels, and adding the main menu.

Marketing Strategy

If I promoting the game and gain users I would simply put it on both apple app store on iOS and google playstore on android and to gain users I would simply put the game as a free game as an in-app purchases where many users can play the game but still have to watch ads.
